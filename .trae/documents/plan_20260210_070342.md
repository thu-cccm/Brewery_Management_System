# Docker 从入门到精通实战行动方案 (基于当前项目)

这个方案将带领你利用现有的 `Brewery_Management_System` (基于 RuoYi 框架) 项目，一步步实践 Docker 的核心概念。

## 阶段一：后端容器化 (Docker 基础与 Java 应用)
**目标**：将 `ruoyi-admin` 后端模块打包成 Docker 镜像并独立运行。
**核心知识点**：`Dockerfile` 编写、镜像构建 (`docker build`)、容器运行 (`docker run`)。
**执行步骤**：
1.  在项目根目录创建 `Dockerfile`。
2.  使用多阶段构建：
    *   **Build 阶段**：使用 Maven 镜像编译项目，生成 jar 包。
    *   **Run 阶段**：使用 OpenJDK 镜像运行 jar 包。
3.  构建镜像并尝试启动 (需连接本地或临时数据库)。

## 阶段二：前端容器化 (静态资源与 Nginx)
**目标**：将 `ruoyi-ui` 前端模块打包并在 Docker 中运行。
**核心知识点**：多阶段构建 (Node.js + Nginx)、Nginx 配置、反向代理概念。
**执行步骤**：
1.  在 `ruoyi-ui` 目录下创建 `Dockerfile`。
2.  **Build 阶段**：使用 Node.js 环境安装依赖 (`npm install`) 并构建生产环境代码 (`npm run build:prod`)。
3.  **Run 阶段**：将构建好的 `dist` 目录复制到 Nginx 容器中，并编写自定义 `nginx.conf` 处理路由和 API 转发。

## 阶段三：服务编排 (Docker Compose 集成环境)
**目标**：一键启动整个生态系统 (后端 + 前端 + MySQL + Redis)。
**核心知识点**：`docker-compose.yml`、服务依赖 (`depends_on`)、容器网络、环境变量注入。
**执行步骤**：
1.  在项目根目录创建 `docker-compose.yml`。
2.  定义 4 个服务：
    *   `ruoyi-mysql`: 数据库服务 (需初始化 SQL 脚本)。
    *   `ruoyi-redis`: 缓存服务。
    *   `ruoyi-admin`: 依赖 MySQL 和 Redis。
    *   `ruoyi-ui`: 依赖后端服务。
3.  配置服务间的网络通信，确保后端能连接到容器内的数据库。

## 阶段四：进阶优化 (数据持久化与生产配置)
**目标**：模拟生产环境部署，确保数据安全和性能。
**核心知识点**：Docker Volume (数据卷)、Bind Mount (挂载配置)、镜像瘦身 (Alpine Linux)。
**执行步骤**：
1.  **数据持久化**：将 MySQL 数据和 Redis 数据挂载到宿主机，防止容器删除后数据丢失。
2.  **配置挂载**：将 Nginx 配置文件和后端日志挂载出来，方便修改和查看。
3.  **网络优化**：配置 Nginx 反向代理，解决跨域问题，统一对外端口。
